3. Abstract (100-300 words)
PALEO is an agentic AI system for dinosaur-themed survival gameplay. The project addresses a key problem in game AI: many non-player behaviors are predictable, reactive, and disconnected from an interpretable internal state. We aim to build a dinosaur behavior agent that reads gameplay frames, infers high-level behavioral state (for example threat awareness, resource urgency, and risk posture), generates natural-language thought logs, and proposes or executes actions aligned with those inferred needs.

This is important for both game design and accessibility. For developers, it offers a stress-testing tool to evaluate map balance and resource placement using less-scripted behavior patterns. For players, especially those who struggle with high-input micromanagement, it can assist with survival loops and improve playability.

Our technical approach combines computer vision, a trained behavior classifier, and Letta-based agent orchestration. Based on our planning notes, we will use public wildlife behavior datasets (Snapshot Serengeti plus a Kaggle predator/prey dataset) to train a lightweight classifier that maps visual cues to behavioral intent classes. In deployment, each dinosaur is represented by its own Letta agent instance with memory blocks for individual state (identity, current needs, recent events, risk history), while shared game knowledge is retrieved separately. The novel element is combining interpretable thought logs with behavior-conditioned action decisions, so outputs are both actionable and explainable.

6. Models to Use
Baseline model(s):
- Rule-based script using OpenCV.

Pretrained backbone:
- ResNet-18 (single final backbone).

Fine-tuning strategy:
- Fine-tune final classification head on selected animal behavior classes.
- Evaluate class-weighted loss and augmentation variants for imbalance handling.
- Feed classifier predictions into a Letta-hosted agent decision policy that writes thought logs and actions.

Why these models are appropriate:
The OpenCV baseline provides a transparent control reference, ResNet-18 is lightweight and stable for transfer learning on limited hardware, and Letta provides memory plus a stateful multi-agent interface needed for one-agent-per-dinosaur behavior.

7. Target Customers / Users
Who benefits:
- Game Developers: especially developers of survival games such as ARK who need behavioral stress testing for level design and resource routing.
- End Users (Players): players who want assistance with complex survival micromanagement, including those with accessibility needs.

Real-world impact:
The project tests how effectively vision-language style reasoning can translate visual pixels into physical actions in real time while keeping behavior interpretable.

Market need:
Survival games often suffer from predictable NPC behavior; there is demand for emergent, less-scripted agents that feel more alive.

Practical deployment scenario:
A developer runs the agent across map regions; if a hunger-driven dinosaur repeatedly fails to reach water or food, the map flow and resource distribution can be improved.

8. Technical Plan
Training pipeline:
1) Select and download public wildlife behavior datasets.
2) Build preprocessing and unified label pipeline.
3) Train baseline OpenCV/rule policy and supervised ResNet-18 classifier.
4) Integrate model outputs with PALEO agent loop in Letta (perceive -> decide -> act -> remember).
5) Run controlled evaluation and document failure modes.

Evaluation metrics:
- Classification metrics: accuracy, macro-F1, confusion matrix.
- Agent-level metrics: action success rate, decision latency, and qualitative behavior realism.
- Robustness: performance under class imbalance and visual noise.

Required experiment outputs:
- Convergence curves (training/validation loss and accuracy over epochs).
- Hyperparameter sensitivity (learning rate, augmentation strength, model backbone).
- Baseline comparison table (OpenCV rule baseline vs trained classifier variants).
- Qualitative examples (success and failure cases with short analysis).

Hardware needed:
- Development PC with modern CPU/GPU.
- Sufficient RAM for preprocessing, training, and local inference.
- Optional GPU acceleration for faster fine-tuning.
- Letta runtime for stateful per-dinosaur agents.

Timeline (Week-by-week):

- Week 2: Train first supervised baseline and OpenCV baseline.
- Week 3: Hyperparameter sweep + sensitivity analysis.

10. Challenges Encountered
- Data imbalance: public wildlife data is long-tailed across species/behaviors.
- Overfitting: transfer from wildlife imagery to game scenarios may overfit to source-domain cues.
- GPU/compute constraints: training and real-time inference must fit student hardware limits.
- Label noise: public dataset labels may be inconsistent across sources.
- Model instability: action policy may oscillate when confidence is low.

How we plan to address them:
- Use class balancing, weighted loss, and targeted sampling.
- Add simple domain adaptation with optional self-recorded gameplay examples.
- Use confidence thresholds and fallback safe actions in the agent loop.
- Restrict full automation to private/testing environments and keep an advisor-only mode for safer deployment.
- Document ethics and platform constraints (no unfair online play or anti-cheat evasion).
